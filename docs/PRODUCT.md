# Product vision — HUB Contábil OS

## Problema

Escritórios brasileiros usam 5–8 ferramentas (XML + guias + tarefas + WhatsApp + portal + apuração + pagamento). O HubStrom integra bem o operacional, mas **não fecha** apuração assistida + pagamento automático + firm P&L + reforma tributária.

## Solução

Um **Contábil OS** com três moats:

1. **Fiscal Autopilot** — pipeline ponta a ponta com humano no loop
2. **Firm OS + Client OS** — operação interna e experiência do cliente no mesmo sistema
3. **Reforma-ready** — CBS/IBS nativo (camada futura)

## Persona primária

Sócio / coordenador fiscal de escritório com 50–500 clientes.

## Job to be done

"Quero que o DAS do mês saia sem caça a XML, sem planilha de status e sem cliente ligando pedindo a guia — e que eu saiba se foi paga."

## Princípios

- Um evento, um pipeline — não módulos soltos
- Aprovação humana nos estágios críticos (apurar / pagar)
- Toda feature aponta para um benchmark (evita feature soup)
- Piloto estreito (DAS) antes de cobertura ampla
